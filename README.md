# Zoo Ticketing System - Database Project

## 1. Project Overview
This project is a Zoo Ticketing System database. It manages visitors, transactions, guided tours, employees, and memberships.

The system follows a top-down design approach:
1. System design (UI screens)
2. ERD design
3. Database implementation
4. Data generation and backup

---

## 2. System Description
The system allows:
- Visitors to purchase tickets
- Booking guided tours
- Employees to process transactions
- Tracking memberships
- Managing zoo operations data

---

## 3. Database Design (In Progress)
The database is currently being designed using ERD.

Expected main entities:
- Visitors
- Transactions
- Transaction_Items
- Employees
- Memberships
- Guided_Tours

Relationships and constraints will be finalized after ERD completion.

---

## 4. Data Generation Methods
The project will include 3 data generation methods:
1. Manual SQL INSERT statements
2. Mockaroo generated data
3. Python script for large-scale data (20,000+ rows)

---

## 5. Backup Plan
A full database backup will be created and tested on another machine to ensure successful restoration.

---

## 6. Project Structure
- Stage1 contains SQL scripts and data generation setup
- ERD and DSD will define final schema
